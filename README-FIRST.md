### Git設定（HTTPS接続推奨）

コンテナ環境では、SSH鍵がコンテナの再起動時に消える可能性があるため、HTTPS接続を推奨します。

```bash
# HTTPS接続の設定（推奨）
git remote set-url origin https://github.com/Hiiro-Uchiyama/statutes-rags.git

# 初回push時に認証情報をキャッシュ
git config --global credential.helper store
```

SSH接続を使用する場合は、鍵を永続ボリューム（~/work/.ssh など）に配置してください。

### 技術的な課題と検討事項

1. **インデックス構築の計算量**
   - 全10,435ファイルのインデックス化は計算コストが高い
   - 小規模実験では`--limit`オプションで処理件数を制限することを推奨
   - 詳細は[docs/03-USAGE.md](docs/03-USAGE.md)を参照

2. **評価精度**
   - 初期実験: gpt-oss:20b + 部分インデックスで20問中6問正解（30%）
   - 論文ベースの期待値: 適切なRAG構築で80-90%の精度が見込める
   - 改善の方向性: コンテキスト最適化、モデルの選択、ハイブリッド検索の調整

3. **今後の方向性**
   - [docs/proposals/](docs/proposals/)に提案ドキュメントを配置
   - Agentic RAG、MCP Server統合、評価フレームワーク拡張などを検討中
